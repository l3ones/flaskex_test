 Для теста я использовал виртуальную машину на YandexCloud.

1. Установив все необходимые пакеты и зависимости, я запустил приложение.
Получил ошибку:

"AttributeError: module 'wtforms.validators' has no attribute 'required'"

Модуль "wtforms.validators" не имеет атрибута "required" из-за установленной версии WTForms.
Если используется старая версия WTForms, то нужно заменить в файле forms.py "required" на "InputRequired".

2. После исправления и повторного запуска отобразилась страница на HTML и от приложения поступил долгожданный ответ 200.

127.0.0.1 - - [02/Apr/2023 09:14:49] "GET / HTTP/1.1" 200 -

  

   Dockerfile.

1. Используем Python 3.8 в слегка урезанной версии, чтобы уменьшить размер образа.

2. Создаем переменную окружения FLASK_APP, которая сообщает Flask, что файл app.py содержит наше приложение.

3. Создаем рабочую директорию в /app.

4. Копируем файл с зависимостями.

5. Устанавливаем зависимости из файла requirements.txt. Не используем кэш, чтобы уменьшить размер образа.

6. Копируем все файлы из текущей директории в /app внутри контейнера.

7. Приложение будет слушать порт 5000.

8. Указываем команду, которая будет запускаться при запуске контейнера.


   Docker compose.

1. Указываем версию.

2. Создаем разде services, где будем указывает все наобходимые сервисы для развертывания.

3. Создаем сервис flask_app.

4. Создаем образ находящийся в текущей директории.

5. Связываем порт хоста с портом контейнера.
